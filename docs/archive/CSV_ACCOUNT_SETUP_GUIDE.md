# アカウント情報収集・CSV作成 完全ガイド

このガイドでは、CSV一括インポートに必要な全ての情報を取得する方法を詳しく解説します。

## 📋 CSVフォーマット

```csv
screen,password,mail,mailpw,二段階認証コード取得URL,バックアップコード
```

---

## 🔴 項目の取得方法

### 1. screen（ユーザー名）

**取得方法:**
1. X (Twitter) にログイン
2. 自分のプロフィールページにアクセス
3. URLの `@` 以降の部分がユーザー名
   - 例: `https://x.com/example_user` → `example_user`
4. または、プロフィール画面の表示名の下に `@ユーザー名` が表示されています

**注意事項:**
- `@` マークは **含めません**
- 大文字小文字を区別します

**例:**
```
example_user
```

---

### 2. password（Twitterパスワード）

**取得方法:**
- アカウント作成時に設定したパスワード
- パスワードを忘れた場合は、以下の手順でリセット:
  1. ログイン画面で「パスワードを忘れた」をクリック
  2. メールアドレスまたは電話番号を入力
  3. 送信された認証コードを入力
  4. 新しいパスワードを設定

**セキュリティ推奨:**
- 12文字以上
- 大文字・小文字・数字・記号を含む
- 他のサービスと同じパスワードを使用しない

**例:**
```
MyStr0ng!Pass2024
```

---

### 3. mail（メールアドレス）

**取得方法:**
1. X (Twitter) にログイン
2. 設定 → アカウント → アカウント情報 → メールアドレス
3. または、アカウント作成時に登録したメールアドレス

**確認方法:**
- X (Twitter) から届いた通知メールを確認

**対応メールプロバイダー:**
- Gmail
- Outlook (Hotmail, Live.com)
- Yahoo Mail
- iCloud Mail
- その他のIMAPサポートメール

**例:**
```
example@gmail.com
```

---

### 4. mailpw（メールパスワード）

メールアカウントのパスワード（メール認証コード自動取得に必須）

#### 📧 **Gmail の場合**

**アプリパスワードの作成（必須）:**

1. **Google アカウントにログイン**
   - https://myaccount.google.com/ にアクセス

2. **2段階認証を有効化**（まだの場合）
   - 「セキュリティ」→「2段階認証プロセス」
   - 画面の指示に従って設定

3. **アプリパスワードを生成**
   - 「セキュリティ」→「アプリパスワード」
   - アプリを選択: 「メール」
   - デバイスを選択: 「その他（カスタム名）」
   - 名前を入力: 「XLO System」
   - 「生成」をクリック
   - **16桁のパスワードが表示されます**（スペースなしで記録）

4. **CSVに記入**
   ```
   例: abcdefghijklmnop
   ```

**注意:**
- 通常のGoogleパスワードではなく、必ず **アプリパスワード** を使用
- アプリパスワードは一度しか表示されないので、必ず保存

---

#### 📧 **Outlook/Hotmail の場合**

**アプリパスワードの作成:**

1. **Microsoft アカウントにログイン**
   - https://account.microsoft.com/ にアクセス

2. **セキュリティ設定を開く**
   - 「セキュリティ」→「セキュリティの詳細オプション」

3. **2段階認証を有効化**（まだの場合）
   - 「2段階認証」→「オンにする」

4. **アプリパスワードを作成**
   - 「アプリパスワード」→「新しいアプリパスワードの作成」
   - 生成されたパスワードをコピー

5. **CSVに記入**
   ```
   例: abcd-efgh-ijkl-mnop
   ```

---

#### 📧 **Yahoo Mail の場合**

**アプリパスワードの作成:**

1. **Yahoo アカウントにログイン**
   - https://login.yahoo.com/ にアクセス

2. **アカウントセキュリティページを開く**
   - https://login.yahoo.com/account/security

3. **アプリパスワードを生成**
   - 「アプリパスワードの生成」をクリック
   - アプリ名を入力: 「XLO System」
   - 「生成」をクリック
   - 表示されたパスワードをコピー

4. **CSVに記入**
   ```
   例: abcdefghijklmnop
   ```

---

#### 📧 **iCloud Mail の場合**

**アプリ専用パスワードの作成:**

1. **Apple ID アカウントページにアクセス**
   - https://appleid.apple.com/

2. **サインイン**
   - Apple ID とパスワードでサインイン
   - 2ファクタ認証コードを入力

3. **アプリ専用パスワードを生成**
   - 「セキュリティ」セクション
   - 「アプリ用パスワード」→「パスワードを生成」
   - ラベルを入力: 「XLO System」
   - 生成されたパスワードをコピー

4. **CSVに記入**
   ```
   例: abcd-efgh-ijkl-mnop
   ```

---

### 5. 二段階認証コード取得URL

2段階認証（2FA）を自動化するためのURL。

#### 📱 **方法1: 認証アプリを使用（Google Authenticator, Authy等）**

**シークレットキーの取得:**

1. **X (Twitter) で2段階認証を設定**
   - 設定 → セキュリティとアカウントアクセス
   - セキュリティ → 2要素認証
   - 「認証アプリ」を選択

2. **QRコードの代わりにシークレットキーを表示**
   - 「QRコードが読み取れない場合」または「代わりにコードを入力」をクリック
   - 表示される文字列（32文字程度の英数字）をコピー
   - 例: `JBSWY3DPEHPK3PXP`

3. **TOTP生成サービスのURLを作成**
   ```
   otpauth://totp/Twitter:example_user?secret=JBSWY3DPEHPK3PXP&issuer=Twitter
   ```

4. **または、2FA APIサービスを使用**
   - 2FA.live: `https://2fa.live/tok/YOUR_SECRET_KEY`
   - TOTP.app: `https://totp.app/api/token?secret=YOUR_SECRET_KEY`

**CSVに記入例:**
```
otpauth://totp/Twitter:example_user?secret=JBSWY3DPEHPK3PXP&issuer=Twitter
```

または

```
https://2fa.live/tok/JBSWY3DPEHPK3PXP
```

---

#### 📱 **方法2: 既存の認証アプリからエクスポート**

**Google Authenticator の場合:**
1. アプリを開く
2. 右上のメニュー → アカウントのエクスポート
3. エクスポートするアカウントを選択
4. QRコードが表示される
5. QRコードスキャナーでスキャンしてURLを取得

**Authy の場合:**
1. Authyはエクスポート機能がないため、シークレットキーの再取得が必要
2. Twitter で2段階認証を一旦解除し、再設定してシークレットキーを取得

---

#### 📱 **方法3: 2FAバックアップから復元**

もし2段階認証のシークレットキーを保存していない場合:
1. Twitter で2段階認証を一旦 **解除**
2. 再度2段階認証を **設定**
3. その際にシークレットキーを記録

**警告:** 2段階認証を解除すると、一時的にアカウントのセキュリティが低下します。

---

#### 📱 **方法4: 空欄にする（バックアップコードのみ使用）**

もし2FAのシークレットキーが取得できない場合は、この項目を空欄にして、バックアップコードのみを使用することも可能です。

**CSVに記入例:**
```
（空欄）
```

---

### 6. バックアップコード

2段階認証が利用できない場合の代替手段。

**取得方法:**

1. **X (Twitter) にログイン**
2. **設定を開く**
   - 設定 → セキュリティとアカウントアクセス
   - セキュリティ → 2要素認証

3. **バックアップコードを生成**
   - 「バックアップコード」をクリック
   - 「新しいコードを生成」をクリック
   - **10個の8桁コードが表示されます**

4. **CSVに記入**
   - すべてのコードをハイフン（`-`）で区切って記入
   - または改行で区切る（ダブルクォートで囲む）

**形式例:**
```
12345678-23456789-34567890-45678901-56789012-67890123-78901234-89012345-90123456-01234567
```

または（改行を含む場合はダブルクォートで囲む）

```
"12345678
23456789
34567890
45678901
56789012
67890123
78901234
89012345
90123456
01234567"
```

**注意:**
- バックアップコードは **1回だけ使用可能**
- 使用後は新しいコードを生成してください
- 複数のコードを保存しておくことを推奨（最低3つ以上）

## 🔧 CSV作成のベストプラクティス

### 1. Excelで作成する場合

1. **新規Excelファイルを作成**
2. **1行目にヘッダーを入力**
   ```
   screen | password | mail | mailpw | 二段階認証コード取得URL | バックアップコード
   ```
3. **2行目以降にデータを入力**
4. **保存方法:**
   - ファイル → 名前を付けて保存
   - ファイルの種類: `CSV UTF-8（カンマ区切り）(*.csv)`
   - 保存

### 2. Googleスプレッドシートで作成する場合

1. **新規スプレッドシートを作成**
2. **1行目にヘッダーを入力**
3. **2行目以降にデータを入力**
4. **ダウンロード方法:**
   - ファイル → ダウンロード → カンマ区切りの値（.csv、現在のシート）

### 3. テキストエディタで作成する場合

1. **メモ帳またはテキストエディタを開く**
2. **以下の形式で入力**
   ```
   screen,password,mail,mailpw,二段階認証コード取得URL,バックアップコード
   example_user,pass123,mail@example.com,mailpass,url,codes
   ```
3. **UTF-8形式で保存**
   - ファイル → 名前を付けて保存
   - 文字コード: UTF-8
   - 拡張子: .csv

## ⚠️ 注意事項

### セキュリティ

1. **CSVファイルの保護**
   - パスワードや認証情報が含まれるため、厳重に管理
   - ファイルを暗号化することを推奨
   - 使用後は安全に削除

2. **アプリパスワードの使用**
   - Gmail, Outlook, Yahoo等は必ずアプリパスワードを使用
   - 通常のパスワードでは認証エラーになります

3. **2段階認証**
   - 2段階認証を有効にすることでセキュリティが向上
   - バックアップコードは必ず保存

### データ形式

1. **カンマを含むデータ**
   - データ内にカンマが含まれる場合はダブルクォートで囲む
   - 例: `"12345,67890"`

2. **改行を含むデータ**
   - バックアップコードなど、改行を含む場合はダブルクォートで囲む
   - または、ハイフンで区切る

3. **空白**
   - 前後の空白は自動的に削除されます
   - データ内の空白は保持されます

4. **文字コード**
   - 必ず **UTF-8** で保存
   - Shift-JISなどでは日本語が文字化けします

5. **空欄の項目**
   - 2FA URLやバックアップコードは空欄でも構いません
   - ただし、カンマは必ず必要です
   - 例: `user,pass,mail,mailpw,,` （最後の2項目が空欄）

## 🚀 インポート手順

### 1. CSVファイルの準備
- 上記の手順で全ての情報を収集
- CSV形式で保存（UTF-8）

### 2. システムでインポート
1. メインアカウント / スパムアカウント / フォローアカウント管理画面を開く
2. 「CSVインポート」ボタンをクリック
3. CSVファイルを選択
4. プレビューで内容を確認
5. 「インポート」をクリック

### 3. 確認
- インポート結果を確認
- エラーがある場合は、該当アカウントの情報を修正して再インポート

## 🆘 トラブルシューティング

### Q: メールパスワードでエラーが出る

**A: アプリパスワードを使用していますか？**
- Gmail, Outlook, Yahoo等は通常のパスワードでは認証できません
- 必ずアプリパスワードを生成して使用してください

### Q: 2段階認証コード取得URLがわからない

**A: 以下のいずれかの方法を試してください:**
1. 認証アプリのシークレットキーを取得
2. バックアップコードのみを使用（URLは空欄）
3. 2段階認証を再設定してシークレットキーを記録

### Q: CSVが読み込めない

**A: 以下を確認してください:**
1. ファイルがUTF-8で保存されているか
2. ヘッダー行が正しいか（正確に `screen,password,mail,mailpw,二段階認証コード取得URL,バックアップコード`）
3. カンマ区切りまたはタブ区切りになっているか
4. データに不正な改行が含まれていないか（改行を含む場合はダブルクォートで囲む）

### Q: 2FA URLとバックアップコード、どちらを使うべき？

**A: 両方あれば最も確実です:**
- 2FA URLがあれば自動的に6桁コードを生成できます（推奨）
- バックアップコードは2FA URLが使えない時の代替手段
- 理想: 両方を設定
- 最低限: どちらか一方を設定

### Q: バックアップコードが足りなくなったら？

**A: 新しいバックアップコードを生成してください:**
1. Twitter にログイン
2. 設定 → セキュリティ → 2要素認証 → バックアップコード
3. 「新しいコードを生成」をクリック
4. CSVファイルを更新

## 📊 項目の重要度

| 項目 | 重要度 | 用途 | 空欄可否 |
|------|--------|------|----------|
| screen | 🔴 必須 | アカウント識別 | ❌ 不可 |
| password | 🔴 必須 | Twitter/Xログイン | ❌ 不可 |
| mail | 🔴 必須 | メール認証 | ❌ 不可 |
| mailpw | 🔴 必須 | 認証コード自動取得 | ❌ 不可 |
| 二段階認証コード取得URL | 🟡 推奨 | 2FA自動化 | ⭕ 可（バックアップコードがあれば） |
| バックアップコード | 🟡 推奨 | 2FA代替手段 | ⭕ 可（2FA URLがあれば） |

**推奨構成:**
- **最低限:** screen, password, mail, mailpw, バックアップコード
- **推奨:** 全6項目を設定（最も確実）

## 🔐 セキュリティのベストプラクティス

1. **CSVファイルの暗号化**
   - 7-Zip、VeraCrypt等で暗号化して保存
   - パスワード保護されたZIPファイルに入れる

2. **アクセス制限**
   - ファイルのアクセス権限を制限
   - 必要な人のみがアクセスできるようにする

3. **定期的なパスワード変更**
   - 3ヶ月ごとにパスワードを変更
   - CSVファイルも更新

4. **2段階認証の有効化**
   - 全てのアカウントで2段階認証を有効化
   - バックアップコードは別の場所に保管

5. **使用後の削除**
   - インポート完了後、CSVファイルを安全に削除
   - ゴミ箱からも完全に削除
   - または暗号化して保管

## 📝 チェックリスト

- [ ] 各アカウントのscreenが正しい（@なし）
- [ ] メールパスワードがアプリパスワードである
- [ ] 2FA URLまたはバックアップコードが設定されている
- [ ] ファイルがUTF-8で保存されている
- [ ] データに不正な改行や特殊文字が含まれていない
- [ ] テストとして1件だけインポートして動作確認する
